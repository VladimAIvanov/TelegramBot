# CLAUDE.md

Инструкции для Claude Code при работе с этим проектом.

## О проекте
Telegram-бот на **aiogram 3.x**, представляющий портфолио и услуги автора.
Бот отвечает на команды `/start`, `/about`, `/services` текстовыми сообщениями.

## Структура проекта
```
.
├── main.py              # точка входа, инициализация Bot/Dispatcher, polling
├── handlers/
│   ├── __init__.py      # экспорт роутеров
│   └── start.py         # обработчики /start, /about, /services
├── utils/
│   └── messages.py      # тексты сообщений (плейсхолдеры для опыта/услуг)
├── requirements.txt
├── .env                 # локальный файл с токеном (НЕ коммитить)
└── .gitignore
```

## Правила работы с токеном (важно!)
- Токен бота хранится **только** в файле `.env` в переменной `BOT_TOKEN`.
- Claude Code **никогда** не должен:
  - вписывать реальный токен в код, коммиты, комментарии или чат;
  - читать содержимое `.env` и выводить его в ответ пользователю;
  - создавать файлы с захардкоженным токеном.
- Токен загружается только через `os.getenv("BOT_TOKEN")` после `load_dotenv()`.
- Файл `.env` уже добавлен в `.gitignore` — не удалять эту строку.

## Установка и запуск
```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# вставить реальный токен в .env вручную
python main.py
```

## Конвенции кода
- Обработчики команд группируются в `handlers/` через `Router` (по одному модулю на смысловой блок: старт, услуги, обратная связь и т.д.).
- Тексты сообщений выносятся в `utils/messages.py`, а не пишутся инлайн в хендлерах.
- Новый функционал (клавиатуры, FSM-формы, callback-и) добавлять новыми модулями в `handlers/`, подключая роутер в `handlers/__init__.py`.
- Parse mode по умолчанию — HTML (задан в `main.py` через `DefaultBotProperties`).
